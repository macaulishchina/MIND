# Working Conventions

> Record stack-specific conventions here once the project chooses them.

## Status

This file is intentionally minimal in the initial scaffold.
It should grow only when the repo has real conventions worth preserving.

## Default Expectations

- Prefer explicit names over clever abbreviations.
- Keep files and changes small enough to review comfortably.
- Document public behavior when it is not obvious.
- Keep tests deterministic when tests exist.
- Match established local patterns before introducing new ones.

## Add Later

- Tooling commands for linting, formatting, type checks, and tests
- Naming conventions tied to the chosen language or framework
- Module layout guidance once the architecture stabilizes
