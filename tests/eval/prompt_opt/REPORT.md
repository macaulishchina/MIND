# STL 提示词优化与跨模型评测报告

## 1. 概述

本报告记录了 STL（语义翻译层）提取提示词的优化过程和跨模型评测结果。

**目标：** 为 STL v2 语法找到最优的系统提示词，并评估不同 LLM 在实际生产约束下的提取质量与速度表现。

**关键成果：**
- 提示词从 233 行压缩至约 90 行有效行（131 行含代码框架），平均评分从 8.3 提升至 9.1
- 在 20 个测试用例上完成了 8 个模型的跨模型对比评测（10s 超时约束）
- 确定 gpt-5.4-mini 为生产首选，claude-opus-4-6 为质量标杆

---

## 2. 提示词优化

### 2.1 优化历程

| 轮次 | 变更要点 | 平均分 (claude-opus-4-6) |
|------|----------|------------------------|
| 初始 | 冗长的 233 行提示词，示例与规则交织 | 8.3 |
| R1-R3 | 去除冗余示例，规则化表达 | 8.5 → 8.7 |
| R4-R6 | 谓词种子表整合，修正/撤回互斥规则 | 8.8 → 9.0 |
| R7-R8 | 幽灵事实禁令、correction scope 限定、neg vs retract 边界 | 9.1 |

### 2.2 优化原则

1. **规则驱动 > 示例堆砌** — 把每条期望行为写成明确规则，而非靠大量示例暗示
2. **精简文本** — 90 行/~5KB 是当前甜区；更短会丢规则，更长会分散注意力
3. **互斥条件显式声明** — `correct_intent` / `retract_intent` 的互斥，`neg` vs `retract` 的边界
4. **禁止项比允许项更有效** — "NEVER output the old/wrong fact" 比 "only output new facts" 效果好
5. **结构化章节** — Entities / Statements / Predicates / Corrections / Output rules 清晰分层

### 2.3 最终提示词

文件：`mind/stl/prompt.py`（131 行 / 5980 字节）

核心规则：
- `$id` 必须先定义再引用
- 实体标签用专有名词或亲属称谓，不用描述
- 关系谓词参数用 `@ref`（人/地），不用 `$ref`
- 动作谓词含直接宾语（`resign(@p, "org")`），不拆分
- `correct_intent` 只输出新事实，禁止任何旧事实的"幽灵"
- `retract_intent` 仅含否认描述，其余事实正常提取
- 二元事件包含双方 `@ref`

## 4. 评测方法

### 4.1 LLM-as-Judge

使用 `gpt-5.4-nano` 作为评审模型，对每个模型的提取结果进行 7 维度打分（0–10）。

| 维度 | 权重 | 说明 |
|------|------|------|
| completeness | 20% | 所有事实是否完整提取 |
| predicate_choice | 15% | 谓词选择是否语义最优 |
| argument_correctness | 15% | 参数类型和值是否正确 |
| correction_handling | 15% | 修正/撤回是否正确处理 |
| modifier_attachment | 10% | 修饰符是否附着到正确 $id |
| no_hallucination | 15% | 是否编造了未提及的事实 |
| format_compliance | 10% | 输出格式是否严格合规 |

加权平均得到单次质量分，再对所有用例取均值得到模型质量分。

### 4.2 综合评分公式

综合得分使用 min-max 归一化后的加权公式：

$$\text{total} = 0.25 \times \text{speed} + 0.35 \times \text{quality} + 0.25 \times \text{reliability} + 0.15 \times \text{throughput}$$

- **speed:** 基于平均延迟，min-max 归一化（越快越高）
- **quality:** 基于加权质量均分，min-max 归一化
- **reliability:** 成功率 = ok / total
- **throughput:** 基于总语句数 / 总时间，min-max 归一化

### 4.3 约束条件

- **超时：** 每个请求 10 秒硬超时（ThreadPoolExecutor 实现）
- **并发：** 每模型串行，模型间并行
- **提供者：** 全部通过 leihuo（ai.leihuo.netease.com）代理，OpenAI 兼容协议
