> **版本**：v2.0  **日期**：2026-04-01
> **状态**：设计定稿，已指导实现。

## 1. 目标与动机

### 要解决的问题

当前 MIND 的提取流程是：

```
自然语言对话
  → LLM 提取自然语言事实 [{text, confidence}]
  → LLM 归一化为 FactEnvelope (JSON)
  → 写入向量库 + 关系库
```

这有三个结构性缺陷：

1. **两步 LLM 调用**（提取 + 归一化），成本高、信息在中间步骤丢失
2. **JSON 输出脆弱**，格式错误导致整条丢失
3. **只能表达事实**，希望、条件、转述、谎言、不确定性等信息被丢弃

### 语义翻译层的三个目标

| # | 目标 | 策略 |
|---|------|------|
| 1 | **不丢信息** | 引入态度/条件/转述语义，覆盖事实之外的信息层 |
| 2 | **压缩 token** | 紧凑的 3+1 行类型语法，比 JSON 输出减少 ~40-50% token |
| 3 | **直接映射到结构化存储** | 每条语句用正则解析，三级容错保证不丢信息 |

### 一句话定义

> 不是直接把自然语言变成"记忆文本"，而是先变成一种**紧凑的语义符号层**。LLM 一步完成翻译，下游用正则完成解析入库。

## 2. 设计原则：语法封闭 × 词汇受控

| 维度 | 策略 |
|------|------|
| **语法结构** | 只有 **3 种语句 + 1 种注释**，解析器用正则完成 |
| **词汇** | 提供种子词表（85 词），LLM **只能使用已有词**，但可建议新词（`:suggested_word`） |

为什么这样设计：

- **语法封闭** → 解析器永远稳定，不需要随词汇更新而更新
- **词汇受控** → 谓词总是已知的，LLM 建议不影响解析逻辑
- **建议机制人类可控** → 审查 `suggested_pred` 后批量晋升为种子词

### v2 三条设计约束

| 约束 | 策略 |
|------|------|
| **LLM 少 token** | 3+1 行类型、无冗余元信息（不输出置信度/溯源） |
| **LLM 少犯错** | 禁止嵌套、禁止列表、arg 只有 4 种原子类型 |
| **LLM 浅思考** | 不要求 LLM 做分类决策（scope、prop/frame、confidence） |

---

## 3. 三类一等公民

| 对象 | 符号 | 含义 | 类比 |
|------|------|------|------|
| **Ref** | `@id` | 实体引用——"谁/什么" | KG 中的 node |
| **Stmt** | `$id` | 语义断言——"什么事/什么属性/什么态度" | KG 中的 edge / reified edge |
| **Note** | `note()` | 自由文本兜底——"无法形式化的信息" | 补充注释 |

关键设计点：

- **实体和关系分开**：`Tom` 是 Ref，`friend(@self, @tom)` 是 Stmt
- **关系、属性、态度语法统一**：`friend(a, b)` 和 `believe(a, $p)` 结构相同
- **类别由词表后标注**：LLM 不决定 prop/frame/qualifier，解析器入库时查词表自动标注
- **证据是系统侧**：不由 LLM 输出，通过 batch_id 关联原始对话自动推断

---

## 4. 3+1 种行类型

每条输出**必须且只能**是以下 4 种之一：

| # | 名称 | 语法 | 用途 |
|---|------|------|------|
| ① | **REF** | `@id: TYPE "key"` | 声明一个实体 |
| ② | **STMT** | `$id = pred(arg, ...)` | 断言一条语义关系 |
| ③ | **NOTE** | `note($id, "text")` | 自由文本兜底 |
| ④ | **COMMENT** | `# ...` | 注释，解析器忽略 |

空行被忽略。不存在其他行类型。

**v1 中的 5 种句式（REF, PROP, FRAME, EV, NOTE）简化为 3 种**：
- PROP 和 FRAME 统一为 STMT（靠词表 category 区分）
- EV 删除（置信度/溯源由系统侧推断）

## 5. 实体引用（REF）详解

### 语法

```
@id: TYPE "key"       # 有名实体
@id: TYPE             # 未命名实体
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 本批次内该实体的局部标识符 |
| `TYPE` | 是 | 实体类型（见种子值） |
| `key` | 否 | 实体原文名称。省略表示未命名实体 |

### `@self`

- `@self` 是保留实体引用，代表 owner（用户本人）
- **不需要声明**，在 STMT 的 arg 中直接使用
- **不能被重新声明**（`@self: person "me"` 是非法的）
- 除 `@self` 外，所有 `@id` 必须通过 REF 行声明后才能使用

### TYPE 种子值

```
person  place  org  brand  event  object  animal  food  concept  time
```

TYPE 不可自创。如果实体不属于任何种子 TYPE，使用 `concept`。

### key 的语义

key 保留对话中的原文表达：
- 中文名用中文：`@mom: person "妈妈"`
- 英文名用英文：`@tom: person "tom"`
- 品牌名用原文：`@ikea: brand "IKEA"`

key 是同一 TYPE 下实体去重的依据。

### 示例

```
@tom: person "tom"          # Tom
@tokyo: place "tokyo"       # 东京
@google: org "Google"       # Google
@p1: person                 # 未命名实体（"我有个朋友"）
```

## 6. 语义断言（STMT）详解

### 语法

```
$id = pred(arg, ...)[:suggested_word]
```

### 参数类型（4 种原子类型）

| 类型 | 语法 | 示例 | 解析器判定 |
|------|------|------|-----------|
| 实体引用 | `@IDENT` | `@tom`、`@self` | 以 `@` 开头 |
| 命题引用 | `$IDENT` | `$p1`、`$f1` | 以 `$` 开头 |
| 字符串 | `STRING` | `"football_player"` | 以 `"` 开头 |
| 数值 | `NUMBER` | `42`、`0.8` | 以数字或 `-` 开头 |

**不允许**其他 arg 形态：

- **禁止内联谓词**：`hope(@self, visit(@self, @tokyo))` 是非法的
- **禁止列表**：`speak(@self, ["中文", "英语"])` 是非法的

要表达嵌套语义，先声明中间 `$id`：

```
$p1 = visit(@self, @tokyo)
$f1 = hope(@self, $p1)
```

要表达多值，展开为多条 STMT：

```
$p1 = speak(@self, "中文")
$p2 = speak(@self, "英语")
```

### 谓词建议 `:suggested_word`

当种子词表中没有精确匹配的谓词时，LLM **必须使用最接近的种子词**作为
`pred`，同时可在 `)` 之后用 `:` 附加建议词：

```
$p1 = friend(@self, @tom):childhood_friend
$p2 = like(@self, "coffee"):obsessed_with
```

规则：

| 规则 | 说明 |
|------|------|
| `pred` 必须是种子词 | 确保解析器永远稳定 |
| `:suggested_word` 是可选的 | 不是每条 STMT 都需要 |
| suggested_word 格式为 `lowercase_snake_case` | 与种子词格式一致 |
| suggested_word 不影响解析和存储语义 | 独立记录，供后续审查 |

### 语义类别（后标注）

v1 中 PROP、FRAME、QUALIFIER 是三种语句类型。v2 中语法统一为 STMT，
类别由**解析器入库时自动标注**（查词表 `category` 字段），LLM 无需感知。

| category | 语义 | 示例谓词 |
|----------|------|---------|
| `prop` | 关于世界的断言 | `friend`, `occupation`, `live_in` |
| `frame` | 对命题的态度/包装 | `believe`, `hope`, `say`, `neg` |
| `qualifier` | 修饰另一条命题 | `time`, `degree`, `quantity` |

### 别名

v2 中别名使用 `alias` 谓词表达（不再是 REF 行参数）：

```
@tom: person "tom"
$p1 = alias(@tom, "小汤")
$p2 = alias(@tom, "Tommy")
```

### 示例

```
$p1 = friend(@self, @tom)
$p2 = occupation(@tom, "football_player")
$p3 = live_in(@tom, @tokyo)
$p4 = time($p3, "2025")
$p5 = resign(@tom)
$p6 = time($p5, "next_month")
$f1 = hope(@self, $p5):hope_stay
$p7 = alias(@tom, "小汤")
```

## 7. 注释（NOTE）详解

### 语法

```
note($target_id, "free text content")
```

### 用途

NOTE 是**信息保全的最后防线**——无法用 REF + STMT 表达的信息放这里。

```
note($p1, "用户说这句话时笑了")
note($f1, "语气是半开玩笑的")
```

### 约束

- NOTE 必须关联一个已存在的 `$id`
- NOTE 的文本内容不参与结构化解析
- NOTE 内部不允许换行
- 如果一段 STL 中大量信息只能靠 NOTE 表达，说明词表需要扩展

## 9. 形式文法（EBNF）

```ebnf
(* ====== 顶层 ====== *)
program       = { line NL } ;
line          = blank | comment | ref_line | stmt_line | note_line ;
blank         = WS* ;

(* ====== COMMENT ====== *)
comment       = WS* "#" { ANY } ;

(* ====== REF ====== *)
ref_line      = "@" IDENT ":" WS+ TYPE ( WS+ STRING )? ;
TYPE          = IDENT ;

(* ====== STMT ====== *)
stmt_line     = "$" IDENT WS* "=" WS* PRED "(" arg_list ")" [ ":" SUGGESTED ] ;
PRED          = IDENT ;
SUGGESTED     = IDENT ;
arg_list      = arg { "," arg } ;
arg           = WS* ( ref_arg | prop_arg | STRING | NUMBER ) WS* ;
ref_arg       = "@" IDENT ;
prop_arg      = "$" IDENT ;

(* ====== NOTE ====== *)
note_line     = "note(" "$" IDENT "," WS* STRING ")" ;

(* ====== 词法 ====== *)
IDENT         = LETTER { LETTER | DIGIT | "_" } ;
STRING        = '"' { CHAR } '"' ;
NUMBER        = [ "-" ] DIGIT { DIGIT } [ "." DIGIT { DIGIT } ] ;
LETTER        = "a".."z" | "A".."Z" ;
DIGIT         = "0".."9" ;
CHAR          = ANY_EXCEPT_UNESCAPED_DQUOTE | '\"' ;
NL            = "\n" | "\r\n" ;
WS            = " " | "\t" ;
```

### 文法特性

| 特性 | 说明 |
|------|------|
| **正则可解析** | 每种行类型都可以用一条正则表达式匹配 |
| **无递归** | arg 不包含嵌套结构，不需要递归下降 |
| **无歧义** | 每行的首字符（`@`/`$`/`n`/`#`/空白）唯一确定行类型 |
| **行独立** | 每行独立解析，单行错误不影响其余行 |

### 硬约束清单

| # | 约束 | 说明 |
|---|------|------|
| C1 | **每行一条语句** | 不允许多条语句在同一行 |
| C2 | **先声明后使用** | `@id` 必须在 REF 行中声明后才能在 STMT/NOTE 中引用 |
| C3 | **`@self` 免声明** | `@self` 是隐式存在的保留实体 |
| C4 | **`@self` 不可重声明** | `@self: ...` 是非法行 |
| C5 | **arg 无嵌套** | arg 只能是 4 种原子类型 |
| C6 | **arg 无列表** | 不存在 `[...]` 语法 |
| C7 | **pred 必须是种子词** | 非种子词只能出现在 `:suggested_word` 位置 |
| C8 | **TYPE 必须是种子值** | 只能使用 §5 中列出的 TYPE |
| C9 | **STRING 无换行** | 字符串内不允许出现换行符 |
| C10 | **$id 本批次唯一** | 同一段 STL 中不能出现两个相同的 `$id` |
| C11 | **note 引用已存在的 $id** | `note($id, ...)` 的 `$id` 必须在之前的 STMT 行中声明 |

## 10. 完整示例集

### 10.1 简单关系 + 属性

**输入**：我朋友 Tom 是足球运动员

```
@tom: person "tom"
$p1 = friend(@self, @tom)
$p2 = occupation(@tom, "football_player")
```

### 10.2 未命名实体

**输入**：我有个朋友是足球运动员

```
@p1: person
$p1 = friend(@self, @p1)
$p2 = occupation(@p1, "football_player")
```

### 10.3 多值属性（展开）

**输入**：我会说中文、英语和一点点日语

```
$p1 = speak(@self, "中文")
$p2 = speak(@self, "英语")
$p3 = speak(@self, "日语")
$p4 = degree($p3, "slight")
```

### 10.4 希望

**输入**：我希望 Tom 来东京

```
@tom: person "tom"
@tokyo: place "tokyo"
$p1 = visit(@tom, @tokyo)
$f1 = hope(@self, $p1)
```

### 10.5 条件

**输入**：如果明天不下雨，我打算去跑步

```
$p1 = event(@self, "rain")
$p2 = neg($p1)
$p3 = time($p2, "tomorrow")
$p4 = plan(@self, "running")
$f1 = if($p3, $p4)
```

### 10.6 转述

**输入**：Mike 说 Tom 住在东京

```
@mike: person "mike"
@tom: person "tom"
@tokyo: place "tokyo"
$p1 = live_in(@tom, @tokyo)
$f1 = say(@mike, $p1)
```

## 11. 数据库映射

### 关系型数据库 Schema

```sql
-- ========== 会话与轮次 ==========

CREATE TABLE conversations (
    id          TEXT PRIMARY KEY,
    started_at  TIMESTAMPTZ DEFAULT NOW(),
    metadata    JSONB DEFAULT '{}'
);

CREATE TABLE turns (
    id          SERIAL PRIMARY KEY,
    conv_id     TEXT NOT NULL REFERENCES conversations(id),
    turn_index  INT NOT NULL,
    role        TEXT NOT NULL,            -- "user" | "assistant"
    content     TEXT NOT NULL,
    created_at  TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE (conv_id, turn_index)
);

CREATE TABLE extraction_batches (
    id          TEXT PRIMARY KEY,
    conv_id     TEXT NOT NULL REFERENCES conversations(id),
    turn_start  INT NOT NULL,
    turn_end    INT NOT NULL,
    model       TEXT,
    created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- ========== 核心语义表 ==========

CREATE TABLE refs (
    id          TEXT PRIMARY KEY,
    owner_id    TEXT NOT NULL,
    scope       TEXT NOT NULL,            -- "self" | "named" | "unnamed"
    ref_type    TEXT,                     -- "person" | "place" | ...
    key         TEXT,
    created_at  TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE statements (
    id             TEXT PRIMARY KEY,
    owner_id       TEXT NOT NULL,
    batch_id       TEXT REFERENCES extraction_batches(id),
    predicate      TEXT NOT NULL,
    category       TEXT,                  -- "prop" | "frame" | "qualifier" (自动标注)
    suggested_pred TEXT,                  -- ":suggested_word" 记录
    superseded_by  TEXT REFERENCES statements(id),
    is_current     BOOLEAN DEFAULT TRUE,
    created_at     TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE stmt_refs (
    stmt_id     TEXT NOT NULL,
    position    INT NOT NULL,
    ref_id      TEXT NOT NULL,
    PRIMARY KEY (stmt_id, position)
);

CREATE TABLE notes (
    id          SERIAL PRIMARY KEY,
    target_id   TEXT NOT NULL,
    content     TEXT NOT NULL,
    created_at  TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE vocab_registry (
    word         TEXT PRIMARY KEY,
    category     TEXT NOT NULL,           -- 语义域名或 "prop"/"frame"/"qualifier"
    arg_schema   TEXT,
    definition   TEXT NOT NULL,
    source       TEXT DEFAULT 'seed',
    usage_count  INT DEFAULT 0,
    status       TEXT DEFAULT 'candidate'
                 CHECK (status IN ('candidate','established','seed','dormant','archived')),
    first_seen_turn INT,
    last_used_turn  INT,
    created_at   TIMESTAMPTZ DEFAULT NOW(),
    last_used    TIMESTAMPTZ
);

CREATE TABLE coreference (
    id          SERIAL PRIMARY KEY,
    source_expr TEXT,
    resolved_to TEXT NOT NULL,
    turn_id     INT REFERENCES turns(id),
    confidence  REAL NOT NULL,
    method      TEXT
);

CREATE TABLE coref_pending (
    id          SERIAL PRIMARY KEY,
    source_expr TEXT,
    candidates  JSONB NOT NULL,
    turn_id     INT REFERENCES turns(id),
    status      TEXT DEFAULT 'pending'
);

CREATE TABLE temporal_specs (
    stmt_id        TEXT PRIMARY KEY REFERENCES statements(id),
    time_type      TEXT CHECK (time_type IN ('point','interval','fuzzy')),
    resolved_start TEXT,
    resolved_end   TEXT,
    fuzzy_desc     TEXT,
    window_days    INT,
    anchor_turn    INT NOT NULL
);
```

### ID 策略

**`$id`（statements）**：解析器生成全局唯一 ID = `{batch_id}_{local_id}`。

**`@id`（refs）**：按 `(owner_id, ref_type, key)` 做 upsert。两次对话都提到 `person "tom"` → 同一行。

### 写入规则

| 句式 | 写入 |
|------|------|
| `@id: TYPE "key"` | `refs` 表 upsert |
| `$id = pred(args)` | `statements` 1 行 + `stmt_refs` N 行；含 `time()` 时同步写 `temporal_specs` |
| `note($id, ...)` | `notes` 表 1 行 |
| `# ...` | 忽略 |
| 共指消解结果 | `coreference` 或 `coref_pending`（由焦点栈模块写入） |
| `correct_intent` / `retract_intent` | 触发修正流程 |

### 典型查询

```sql
-- "Tom 的职业是什么"
SELECT sr.ref_id FROM stmt_refs sr
  JOIN statements s ON sr.stmt_id = s.id
  WHERE s.predicate = 'occupation' AND sr.position = 1
  AND s.id IN (SELECT stmt_id FROM stmt_refs WHERE ref_id = '@tom' AND position = 0);

-- "我所有的朋友"
SELECT sr.ref_id FROM stmt_refs sr
  JOIN statements s ON sr.stmt_id = s.id
  WHERE s.predicate = 'friend' AND sr.position = 1
  AND s.id IN (SELECT stmt_id FROM stmt_refs WHERE ref_id = '@self' AND position = 0);

-- "Tom 说过的所有话"
SELECT s2.* FROM statements s1
  JOIN stmt_refs sr ON s1.id = sr.stmt_id AND sr.position = 1
  JOIN statements s2 ON sr.ref_id = s2.id
  WHERE s1.predicate = 'say'
  AND s1.id IN (SELECT stmt_id FROM stmt_refs WHERE ref_id = '@tom' AND position = 0);
```

## 13. LLM Prompt 模板

```text
You are a semantic extraction engine.
Convert conversations into structured statements using EXACTLY these 3 forms:

  @id: TYPE "key"                           # declare entity
  $id = predicate(arg, ...)[:suggested]     # assert proposition
  note($id, "explanation")                  # free-text note
  # comment                                 # ignored by parser

## Entity refs
  @id: TYPE "key"                # named entity
  @id: TYPE                      # unnamed entity
  @self                          # the owner/user (implicit, never declare)

## TYPE seeds
  person place org brand event object animal food concept time

## Args — EXACTLY 4 atomic types
  @ref    $ref    "literal"    number

  NO inline predicates: hope(@self, visit(@self, @tokyo)) ← ILLEGAL
  NO lists: speak(@self, ["中文", "英语"]) ← ILLEGAL
  Flatten nested semantics with intermediate $ids.
  Expand multi-value to separate statements.

## Predicate suggestion
  When no seed predicate fits precisely, use the closest seed word
  and append :suggested_word after the closing paren:
    $p1 = friend(@self, @tom):childhood_friend

## Common predicates — use ONLY these seed words

RELATIONSHIPS: friend mother father brother sister spouse partner child
  cousin coworker boss mentor student roommate neighbor classmate
  teammate client landlord doctor pet alias

ATTRIBUTES: name age occupation location workplace education nationality
  like dislike habit hobby skill own use speak live_in work_at study_at

ACTIONS: eat drink plan buy visit meet resign marry move start stop
  birthday gift event

ATTITUDES: believe doubt know uncertain hope want intend say recommend
  ask promise emotion decide defer undecided

LOGIC: neg if cause because must permit should lie joke
  retract_intent correct_intent

MODIFIERS (first arg must be $id):
  time degree quantity frequency duration

- One statement per line
- Declare @refs before using them (@self is implicit)
- Pred MUST be a seed word. Suggest better words via :suggested_word

- Do NOT output natural language outside these 3 forms

## Corrections and retractions
  $p_new = new_assertion(...)
```

## 14. 解析器设计

### 行类型识别：4 条正则

```python
import re

RE_COMMENT = re.compile(r'^\s*#')
RE_REF     = re.compile(r'^@(\w+):\s*(\w+)(?:\s+"([^"]*)")?\s*$')
RE_STMT    = re.compile(r'^\$(\w+)\s*=\s*(\w+)\((.+)\)(?::(\w+))?\s*$')
RE_NOTE    = re.compile(r'^note\(\$(\w+),\s*"(.+)"\)\s*$')
```

4 条正则覆盖全部行类型。

### 参数拆分：纯逗号分割

因为 arg 禁止嵌套，参数拆分变为**纯逗号分割**（仅需处理字符串内逗号）：

```python
def split_args(args_str: str) -> list[str]:
    """在顶层逗号处分割参数，仅需处理字符串引号。"""
    result, current, in_string = [], [], False
    for ch in args_str:
        if ch == '"':
            in_string = not in_string
        if ch == ',' and not in_string:
            result.append(''.join(current).strip())
            current = []
            continue
        current.append(ch)
    if current:
        result.append(''.join(current).strip())
    return result
```

不需要括号匹配栈。

### 容错级联（3 级）

每行独立经过三级管线，确保**不丢信息**：

```
级别 1 — 严格解析
  行 → 正则匹配 + 逗号分割 → 成功 → 入库(标记 strict)
                                 ↓ 失败
级别 2 — 模糊修复
  自动修正常见错误 → 重试严格解析 → 成功 → 入库(标记 fuzzy)
  修复规则：
  - 中文引号 ""→ 英文 ""
  - 尾部括号不平衡
  - 多余空白/缺失空白
                                 ↓ 仍失败
级别 3 — 降级为 NOTE
  原始文本 → note($nearest_id, "PARSE_FAIL: 原文") → 入库(标记 fallback)
  信息不丢失，降级保全。
```

**v2 删除了 LLM 修正级别**——语法足够简单，LLM 修正一次额外 API 调用不值得。

### 未声明引用的处理

当 STMT 中引用了未声明的 `@id` 时，解析器自动创建 fallback REF
（type=`concept`, key=null），避免连锁失败。标记 `parse_level=fallback`。

### 关键特性

- **行级独立**：每行独立解析，单行错误不影响其余行
- **纯正则**：不需要括号匹配栈
- **三级容错**：strict → fuzzy → fallback，保证信息不丢
- **解析标签**：每条入库记录标记解析级别，可追溯

## 15. 共指消解：焦点栈

### 问题

中文对话大量省略主语（零指代），长对话中代词"他/她"的指代可能模糊。

### 管线位置：Prompt 注入

焦点栈作为 **LLM 提取前的上下文注入**：

```
每次提取前：
  1. 从 refs + coreference 表查询当前焦点栈 top-k
  2. 将焦点栈注入 Prompt 的上下文区
  3. LLM 在提取时直接使用焦点栈中的 @id

提取完成后：
  4. 更新焦点栈评分
  5. 记录消解事件到 coreference / coref_pending 表
```

注入格式：

```text
## Active entities (focus stack)
The following entities are currently active in conversation context.
Use their @id when they are referenced by pronouns or zero-anaphora:
  @tom: person "tom"    # score=0.82, last mentioned turn 6
  @sarah: person "sarah" # score=0.65, last mentioned turn 8
  @mom: person "妈妈"    # score=0.51, last mentioned turn 15
```

### 评分公式（五维加权）

```
S(e) = 0.35*R + 0.20*F + 0.20*G + 0.15*T + 0.10*P

R 近因性 = 0.78^(当前turn - 最近提及turn)
F 频度   = min(1.0, count/10)
G 语法角色 = 主语1.0 / 宾语0.6 / 其他0.3
T 话题相关 = 当前turn与实体的embedding相似度
P 说话人   = 说话人提过+0.1 / 说话人自身+0.2
```

### 中文零指代处理

句子省略主语时的四步级联：

1. **话题链延续** → 前句主语 = 焦点 top-1？直接继承
2. **动词语义约束** → "辞职"需要 human → 过滤焦点栈中非 human 实体
3. **分差判断** → top-1 vs top-2 分差 ≥ margin → 接受；< margin → 标记歧义
4. **全部失败** → 生成未命名 `@anon: entity` + 写入 `coref_pending`

## 16. 时间语义

### 三种时间类型

| 类型 | 语法示例 | 说明 |
|------|---------|------|
| `point` | `time($p, "2026-04")` | 精确到年/月/日 |
| `interval` | `time($p, "2026-04", "2026-06")` | 起止区间 |
| `fuzzy` | `time($p, "recent")` | 模糊描述 |

### 锚点规则

**相对时间的参考点 = 该 turn 的实际时间戳**。

| 原始表达 | turn 时间 | 解析结果 |
|---------|----------|----------|
| `"next_month"` | 2026-03-30 | `resolved_start = "2026-04"` |
| `"last_year"` | 2026-03-30 | `resolved_start = "2025"` |
| `"recent"` | 2026-03-30 | `fuzzy_desc = "recent"`, `window_days = 30` |

### 存储

解析后写入 `temporal_specs` 表。`time()` qualifier 仍然作为 `statements` 表中的语句存在，`temporal_specs` 是其结构化补充。

## 17. 版本控制与修正

### 问题

用户会修正之前的说法。需要标记旧信息被取代，而不是物理删除。

### 设计约束

每次提取是干净独立的上下文，LLM **不知道**数据库中已有语句的 `$id`。
因此版本控制不能靠 LLM 引用旧 `$id`。

### 修正流程

LLM 表达修正意图，解析器查库匹配：

```
1. LLM 提取 → 标记修正意图：
   $p1 = uncertain(@tom, resign(@tom))
   $f1 = correct_intent(@self, $p1)
   note($f1, "CORRECTION: 修正之前关于 Tom 辞职的说法")

2. 解析器检测 correct_intent/retract_intent → 触发修正流程

3. 用语义描述 + predicate/args 查询数据库 → 匹配旧语句 → 标记：
   旧语句 is_current = FALSE, superseded_by = 新语句 id

4. 找不到匹配 → 新语句作为独立语句入库
```

### 查询约定

- 默认查询加 `WHERE is_current = TRUE`
- 历史回溯可移除此条件，查看修改链
- 通过 `superseded_by` 追溯版本演进

## 18. 与 v1 的差异摘要

| 维度 | v1 | v2 |
|------|-----|-----|
| 行类型 | 5+1 (REF, PROP, FRAME, EV, NOTE, COMMENT) | 3+1 (REF, STMT, NOTE, COMMENT) |
| EV | 必须为每条 $id 附加 ev() | 删除，系统侧推断 |
| REF 语法 | `@id = @local/TYPE("key")` | `@id: TYPE "key"` |
| scope | local / world / blank | 无（统一为 named/unnamed） |
| `@self` | 需要 `@s = @self` 别名 | 直接在 arg 中使用 |
| PROP/FRAME/QUALIFIER | 三种句式 | 统一为 STMT，category 后标注 |
| 内联嵌套 | 允许 2 层 | 禁止 |
| 列表 | `[a, b, c]` 语法 | 禁止，展开为多条 STMT |
| arg 类型 | 6 种 | 4 种原子 |
| alias | REF 行的 `alias=[...]` 参数 | `$id = alias(@ref, "name")` |
| blank node | `_:id` 语法 | `@id: TYPE`（key 省略） |
| 谓词自创 | LLM 自由创造 + NEW_PRED note | 禁止自创，只能 `:suggested_word` |
| 解析器 | 括号匹配栈 + 4 级级联 | 纯逗号分割 + 3 级级联 |
| 解析器修复 | 含 LLM 批量修正（Level 3） | 删除 LLM 修正，语法够简单 |
| 证据表 | DDL 存在，LLM 输出 ev() | DDL 已移除 |

## 19. 已知边界与不追求的事项

### 覆盖范围估计

| 信息层次 | 覆盖率 | 说明 |
|---------|--------|------|
| 事实/属性/关系 | ~95% | 多值展开后基本完备 |
| 态度/意图/义务 | ~90% | attitude/logic 域覆盖 |
| 逻辑/推理 | ~70% | if/cause/neg 覆盖主要场景 |
| 元交互 | ~60% | retract/correct/joke 覆盖常见修正 |
| 共指消解 | 焦点栈 + 零指代规则（§15） |
| 时间语义 | 三类型 + 锚点即时解析（§16） |
| 版本控制 | retract/correct + is_current（§17） |
| 无法形式化的残余 | 100% | note() 兜底 |

### 第一版不追求

| 不追求 | 原因 |
|--------|------|
| 完整一阶逻辑（∀/∃ 量化） | 日常对话极少出现严格量化 |
| 多层嵌套量词 | 复杂度激增，LLM 表达不稳定 |
| 完整时态逻辑 | time() qualifier 已够用 |
| 全局实体消歧 | 由应用层通过 alias 匹配和 embedding 相似度解决 |
| 严格的逻辑完备性 | 目标是信息保全，不是定理证明 |
