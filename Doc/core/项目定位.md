# MIND 项目定位

## 1. 一句话定义

MIND 是一个 AI 记忆质量层（Memory Quality Layer），解决的核心问题不是"让 AI 有记忆"，而是"让 AI 的记忆可靠、可控、长期可用"。

## 2. 项目背景

当前 AI 记忆系统（以 mem0 为代表）已经解决了记忆的基础设施问题：存储、检索、基础增删改查。但在实际长期使用中，这些系统面临三个未解决的结构性问题：

1. LLM 提取的记忆可能是错的，系统没有纠错机制
2. 记忆只增不减，长期运行后质量退化
3. 检索基于语义相似度，不等于基于任务需求

MIND 的目标是在这三个方向上实现突破。

## 3. 我们是什么

- 一个研究驱动的 AI 记忆质量系统
- 聚焦三个核心问题：写入质量、长期健康、任务感知检索
- 面向"个人长期助手"这一个垂直场景
- 追求深度而非广度

## 4. 我们不是什么

- 不是通用的记忆基础设施平台
- 不是 mem0 的替代品
- 不追求支持所有向量库/LLM 的适配广度
- 不追求开箱即用的商业化产品

## 5. 定位：质量层，不是基础设施层

| | mem0 | MIND |
|---|---|---|
| 核心问题 | 怎么存和取记忆 | 怎么让记忆越用越好 |
| 价值主张 | 开箱即用的记忆 API | 长期可靠的记忆质量 |
| 技术深度 | 广而浅（多后端适配） | 窄而深（记忆生命周期） |
| 竞争壁垒 | 生态和用户量 | 记忆质量算法和遗忘机制 |

MIND 不需要替代 mem0。它甚至可以建在 mem0 之上——用现有基础设施做存储和检索，在上层实现记忆质量控制。

## 6. 目标场景

个人长期助手式连续对话：

- AI 记住用户长期偏好，且能区分"确信的"和"推测的"
- AI 记住近期讨论事项，且能随时间自动淡化不再相关的内容
- 当用户问一个复合问题时，AI 能从多个维度召回相关记忆，而不只是找最相似的文本
- 运行半年后，系统的记忆质量不退化

## 7. 战略优势

作为个人研究项目，MIND 的优势不在规模，而在：

1. **可以做研究性探索** — 商业项目不会花时间研究"选择性遗忘"或"信念修正"
2. **可以做深度垂直** — 只服务一个场景，可以把记忆质量做到极致
3. **可以做破坏性设计** — 不受兼容性约束，可以尝试新的记忆表示方式
4. **可以容忍失败** — 试错成本低，可以快速验证和迭代

## 8. 与 mem0 的关系

不是竞争关系，而是分层关系：

```
MIND 层：记忆质量控制（写入验证、遗忘、任务感知检索）
         ↕
基础设施层：存储 + 检索（可以是 mem0，也可以是自建）
```

吸收 mem0 的经验：
- LLM 做决策引擎（不写规则引擎）
- ADD/UPDATE/DELETE/NONE 决策框架作为基线
- Factory 模式支持多后端

保持我们的差异：
- 记忆带置信度和来源追踪
- 更新保留历史（不直接覆盖）
- 遗忘机制（mem0 完全没有）
- 任务感知的多路径检索
