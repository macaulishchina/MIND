# MIND 评测方案

## 1. 目标

验证两件事：
1. MVP 的四个核心操作是否跑通
2. 相比 mem0 基线，我们的增强点是否产生了可观测的价值

## 2. 评测方式

第一版采用人工评测 + 简单脚本统计。

## 3. MVP 必测场景

### 场景 1：稳定偏好写入与召回

步骤：
1. 用户输入"我喜欢黑咖啡"
2. 新会话，询问"你推荐我喝什么"

验证项：
- [ ] 偏好记忆被正确写入
- [ ] 回答中正确使用该偏好
- [ ] 记忆包含 confidence 和 source_context

### 场景 2：偏好更新与版本追踪

步骤：
1. 已存在"喜欢黑咖啡"
2. 用户输入"我现在只喝美式"
3. 再次询问饮品推荐

验证项：
- [ ] 新记忆写入成功
- [ ] 新记忆的 version_of 指向旧记忆
- [ ] 回答优先参考"美式"
- [ ] 通过 history() 可查看变更记录

### 场景 3：最近会话事件召回

步骤：
1. 用户与 AI 完成一次任务讨论
2. 下一次会话问"我们上次讨论到哪里了"

验证项：
- [ ] 事件记忆被正确提取和存储
- [ ] 回答能概括最近讨论内容

### 场景 4：失效记忆不再污染

步骤：
1. 删除一条记忆
2. 发起会触发该记忆的提问

验证项：
- [ ] 该记忆不出现在检索结果中
- [ ] 回答不引用该记忆

## 4. 质量观测指标

以下指标在 MVP 阶段**不作为通过标准**，但需要记录数据以指导 v1.5 优化方向。

### 4.1 写入质量（对应核心问题一）

| 指标 | 定义 | 观测方法 |
|------|------|---------|
| 提取准确率 | 提取的事实中真正正确的比例 | 人工抽检 20 条 |
| 噪声写入率 | 不该记录的信息被错误写入的比例 | 人工抽检 |
| confidence 校准度 | confidence 高的记忆是否确实更准确 | 按 confidence 分桶统计准确率 |

### 4.2 去重质量

| 指标 | 定义 | 观测方法 |
|------|------|---------|
| 重复率 | 同一事实被存为多条记忆的比例 | 人工检查全部记忆 |
| UPDATE 准确率 | 应该判 UPDATE 的是否正确判了 | 人工抽检 |

### 4.3 检索质量（对应核心问题三）

| 指标 | 定义 | 观测方法 |
|------|------|---------|
| 召回命中率 | 需要的记忆是否在 Top-K 中 | 人工构造 10 个测试查询 |
| 任务相关性 | Top-K 中与当前任务真正相关的比例 | 人工评判 |

## 5. 与 mem0 的对比评测

在相同测试场景下，对比 MIND v1 和 mem0 的表现。

### 对比维度

| 维度 | 评测方法 |
|------|---------|
| 提取准确率 | 同一组对话，分别用两个系统提取，人工比对 |
| 重复率 | 运行 50 轮对话后，人工统计重复记忆数量 |
| 可追溯性 | 随机选 5 条记忆，能否追踪到来源对话 |
| 更新正确性 | 10 个偏好变更场景，对比更新结果 |

### 预期差异

| 维度 | mem0 预期 | MIND 预期 |
|------|----------|----------|
| 核心流程 | 可用 | 与 mem0 持平 |
| 可追溯性 | 仅有 SQLite 日志 | confidence + source_context + version_of |
| 遗忘能力 | 无 | v1 无，v1.5 引入 |

MVP 阶段 MIND 与 mem0 在核心流程上预期持平。差异化从 v1.5 开始显现。

## 6. 长期评测方向（v1.5+）

### 对应核心问题一：写入质量

- confidence 降权后的召回准确率变化
- 渐进确认机制的噪声过滤效果

### 对应核心问题二：长期健康

- 运行 N 轮后的记忆污染率
- 时间衰减引入前后的检索质量对比
- 合并压缩后的信息保留率

### 对应核心问题三：任务感知检索

- 查询分解引入前后的召回命中率
- 多路召回 vs 单向量召回的任务完成率

```
检索结果（Top-K）：
最终回答：
是否通过：
问题说明：
```
