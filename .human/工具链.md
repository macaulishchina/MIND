## AI 编码工具配置

### VS Code + GitHub Copilot

项目中已创建 `.github/copilot-instructions.md`，Copilot 会自动读取。

个人设置建议：
```json
{
    "github.copilot.chat.codeGeneration.instructions": [
        { "file": ".ai/CONSTITUTION.md" }
    ]
}
```

### Cursor

项目中已创建 `.cursorrules`，Cursor 会自动读取。

### Claude Code

项目中已创建 `CLAUDE.md`，Claude Code 会自动读取。

### aider

启动时指定规则文件：
```bash
aider --read .ai/CONSTITUTION.md
```

也可以在 `.aider.conf.yml` 中配置：
```yaml
read:
  - .ai/CONSTITUTION.md
```

## 质量工具链

### 一键全量检查

```bash
# 推荐创建别名
alias mindcheck='uv run ruff check mind/ tests/ scripts/ && uv run mypy mind/ tests/ scripts/ && uv run pytest tests/ -x'
```

### Pre-commit（可选但推荐）

如果你想在 commit 前自动检查，可以安装 pre-commit：

```bash
pip install pre-commit
pre-commit install
```

然后在 `.pre-commit-config.yaml` 中配置 ruff + mypy。

## 环境搭建

### 首次设置

```bash
# 1. 克隆项目
git clone <repo-url> && cd MIND

# 2. 安装依赖
uv sync --extra dev --extra api --extra docs --extra mcp

# 3. 启动开发环境
./scripts/dev.sh

# 4. 验证
uv run pytest tests/ -x
```

### 日常开发

```bash
# 启动完整开发环境（API + Worker + Docs + DB）
./scripts/dev.sh

# 只运行测试
uv run pytest tests/ -x

# 检查代码质量
uv run ruff check mind/ tests/
uv run mypy mind/ tests/ scripts/

# 查看文档（如果 dev.sh 已启动）
open http://127.0.0.1:18602
```

## 工具选择原则

1. **已有的不要换**：项目已经配置了 ruff、mypy、pytest、GitHub Actions，用它们就好
2. **AI 工具可以多个并用**：Copilot 做内联补全，Claude Code / aider 做大块任务
3. **标准化优先**：选择团队都能用的工具，而不是某人的私人偏好
4. **配置文件统一**：AI 工具的配置文件都只做一件事——指向 `.ai/CONSTITUTION.md`
